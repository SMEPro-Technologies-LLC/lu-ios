# Blackboard LTI Integration — IOSME

IOSME integrates with Lamar University's Blackboard Learn LMS via **Learning Tools Interoperability (LTI) 1.3** to enable single-sign-on from course pages into the IOSME mobile experience.

---

## LTI Version

IOSME supports **LTI 1.3** with **Advantage extensions** (Deep Linking, Assignment & Grades Services).

---

## Registration

### In Blackboard Admin

1. Log in to Blackboard as system administrator.
2. Navigate to **System Admin → LTI Tool Providers**.
3. Click **Register LTI 1.3/Advantage Tool**.
4. Enter the following details:

| Field | Value |
|-------|-------|
| Client ID | `<LTI_CLIENT_ID>` (generated by Blackboard, store in secret) |
| Tool Name | IOSME |
| Tool Domain | `iosme.lamar.edu` |
| Login Initiation URL | `https://iosme.lamar.edu/lti/login` |
| Tool Redirect URL | `https://iosme.lamar.edu/lti/launch` |
| JWKS URL | `https://iosme.lamar.edu/lti/.well-known/jwks.json` |
| Deep Link Launch URL | `https://iosme.lamar.edu/lti/deep-link` |

5. Save and copy the **Client ID** and **Deployment ID**.

### In IOSME Kubernetes Secret

```bash
kubectl create secret generic iosme-blackboard-lti \
  --namespace iosme-prod \
  --from-literal=client-id='<LTI_CLIENT_ID>' \
  --from-literal=deployment-id='<LTI_DEPLOYMENT_ID>' \
  --from-literal=blackboard-platform-id='https://blackboard.lamar.edu' \
  --from-literal=blackboard-auth-endpoint='https://blackboard.lamar.edu/api/v1/gateway/oauth2/jwttoken' \
  --from-literal=blackboard-jwks-url='https://blackboard.lamar.edu/api/v1/gateway/.well-known/jwks.json'
```

## LTI Launch Flow

```
Student clicks IOSME link in Blackboard course
     │
     ▼
Blackboard sends OIDC login initiation request
→  https://iosme.lamar.edu/lti/login
     │
     ▼
IOSME redirects to Blackboard OIDC authorization endpoint
     │
     ▼
Blackboard authenticates student and issues JWT (id_token)
→  POST https://iosme.lamar.edu/lti/launch
     │
     ▼
IOSME validates JWT signature using Blackboard JWKS
IOSME extracts user claims (email, roles, course context)
IOSME creates/updates local user session
     │
     ▼
Student lands in IOSME with course context pre-loaded
```

---

## Claims Used

| JWT Claim | Purpose |
|-----------|---------|
| `sub` | Blackboard user GUID |
| `email` | User email (linked to Banner CWID) |
| `https://purl.imsglobal.org/spec/lti/claim/roles` | Instructor vs. learner |
| `https://purl.imsglobal.org/spec/lti/claim/context` | Course ID and title |
| `https://purl.imsglobal.org/spec/lti/claim/lis` | Course offering and person source IDs |

## Deep Linking

Instructors can embed specific IOSME content items (assignments, study modules) directly into Blackboard course content areas:

1. Instructor opens a content area in Blackboard.
2. Adds **Web Link/LTI Tool** → selects IOSME.
3. IOSME's Deep Link picker opens.
4. Instructor selects content item and saves.
5. Blackboard stores the content item URL and displays it to students.

---

## Grades Passback (AGS)

IOSME can push assignment grades back to the Blackboard gradebook:

```yaml
# values-prod.yaml
lti:
  ags:
    enabled: true
    scoreEndpoint: "https://blackboard.lamar.edu/api/v1/lti/courses/{courseId}/lineitems"
```

---

## Configuration in Helm Values

```yaml
# values-prod.yaml
lti:
  enabled: true
  version: "1.3"
  secretName: iosme-blackboard-lti
  platformIssuer: "https://blackboard.lamar.edu"
  ags:
    enabled: true
```

## Troubleshooting

| Symptom | Likely Cause | Action |
|---------|-------------|--------|
| "Invalid state parameter" | Session/cookie issue | Clear browser cookies; check IOSME `SESSION_SECRET` |
| JWT signature verification failed | JWKS cache stale | Restart IOSME pod to force JWKS refresh |
| 403 on LTI launch | Deployment ID mismatch | Verify Deployment ID in secret matches Blackboard registration |
| Grade passback failing | AGS token scope missing | Reconfigure Blackboard tool registration to include `https://purl.imsglobal.org/spec/lti-ags/scope/score` |
